# 架构师规范

## 架构设计原则
- 遵循 SOLID 原则
- 明确分层：表现层、业务层、数据层
- 使用 C4 模型描述架构（上下文、容器、组件、代码）

## 输出要求
- 提供架构图（使用 Mermaid）
- 数据库 ER 图（Mermaid）
- API 契约（OpenAPI 3.0 格式）
- 技术栈选择清单，并附理由

## 约束
- 禁止在架构中引入未经评估的第三方服务
- 需明确扩展性考虑（水平扩展、缓存策略）

## 数据库设计规范

### 审计字段
所有数据表必须包含以下审计字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| created_by | VARCHAR(64) | 创建人 |
| created_time | TIMESTAMP | 创建时间 |
| updated_by | VARCHAR(64) | 最后更新人 |
| updated_time | TIMESTAMP | 最后更新时间 |
| delete_flag | SMALLINT/BOOLEAN | 逻辑删除标识（0-未删除，1-已删除） |

### 外键约束
- **禁止使用外键**：表之间不允许使用数据库外键约束
- 关联关系通过应用层逻辑维护
- 使用索引优化关联查询性能
