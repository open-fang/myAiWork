# 测试问题记录

## 问题1: 维护多条答案时主键重复错误
- **发现时间**: 2026-03-27
- **问题描述**: 维护两条答案（每个答案都有中英文内容）保存时报错：`PSQLException: ERROR: duplicate key value violates unique constraint "uk_answer_language"`
- **复现步骤**:
  1. 打开新增题目弹窗
  2. 在可选答案区域点击"+ 新增"添加答案
  3. 添加2行答案条目，每行填写中文答案和英文答案
  4. 点击确定保存
- **预期结果**: 保存成功，答案数据正确存储
- **实际结果**: 后端报错，数据库约束冲突
- **根因分析**:
  - 数据库存在唯一索引 `uk_answer_language` 限制 `(answer_id, language)` 组合唯一
  - 新设计允许多个答案条目(answerItem)使用相同语言
  - 每个答案条目都会在 `questionnaire_answer_text` 表中插入两条记录（中文和英文）
  - 导致多条记录具有相同的 `(answer_id, language)` 组合，违反唯一约束
- **解决措施**:
  - 创建 V4 Flyway 迁移脚本删除 `uk_answer_language` 唯一索引
  - 文件：`V4__fix_answer_language_unique_constraint.sql`
- **状态**: ✅ 已修复

## 问题2: 接口报错但页面提示保存成功
- **发现时间**: 2026-03-27
- **问题描述**: 后端API返回错误时，前端弹窗仍显示"保存成功"
- **复现步骤**:
  1. 触发问题1的场景
  2. 观察前端提示
- **预期结果**: 接口报错时，前端应显示错误信息
- **实际结果**: 前端显示"保存成功"提示
- **根因分析**:
  - 前端 `http` 帮助类使用 `fetch().then(res => res.json())` 解析响应
  - 未检查 HTTP 状态码（`res.ok`）或响应体中的 `code` 字段
  - 即使后端返回 500 错误，`fetch` 也会成功解析 JSON
  - `try-catch` 无法捕获这种情况
- **解决措施**:
  - 修改 `http` 帮助类，检查 `res.ok` 和 `result.code === 200`
  - 不满足条件时抛出 Error，由 catch 块捕获
  - 错误消息从 `result.message` 获取
- **状态**: ✅ 已修复

## 问题3: 弹窗提示方式需改进
- **发现时间**: 2026-03-27
- **问题描述**: 操作结果使用弹窗提示，需要手动关闭
- **复现步骤**: 执行任何保存操作后观察提示方式
- **预期结果**: 使用自动消失的Toast提示气泡
- **实际结果**: 使用alert弹窗，需要手动点击关闭
- **根因分析**:
  - 前端使用 `alert()` 或自定义弹窗组件显示提示
  - 未实现Toast组件
- **解决措施**:
  - 实现 `showToast(message, type)` 方法
  - 添加 Toast 组件到模板
  - 添加 Toast 样式（3秒后自动消失动画）
  - 将所有 `alert()` 调用替换为 `showToast()`
- **状态**: ✅ 已修复
