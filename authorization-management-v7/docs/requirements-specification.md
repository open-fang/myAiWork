# 授权书管理系统 V7 - 需求规格文档

## 1. 项目概述

### 1.1 项目背景
构建一个授权书管理系统，用于管理授权书的基本信息、场景配置、规则配置及发布流程。系统主要服务于授权书管理员，支持授权书的完整生命周期管理。

### 1.2 系统用户
- **授权书管理员**：负责录入授权书基本信息、维护场景和规则、发布授权书

### 1.3 技术栈
- 前端：Vue
- 后端：Java
- 数据库：PostgreSQL

### 1.4 项目目录
- 项目根目录：`/home/pi/aiWork/myAiWork/authorization-management-v7`
- 前端目录：`/authorization-management-v7/frontend`
- 后端目录：`/authorization-management-v7/backend`

## 3. 用户故事清单

### 3.1 授权书列表页模块

#### US-001: 查询授权书列表
**作为** 授权书管理员
**我想要** 通过多种条件查询授权书列表
**以便于** 快速找到目标授权书

**优先级**: Must Have
**故事点**: 5

##### 验收标准
```gherkin
Scenario: 按授权书名称查询
  Given 用户在授权书列表页
  When 用户在"授权书名称"输入框输入"销售授权"
  And 用户点击"查询"按钮
  Then 系统返回名称包含"销售授权"的授权书列表
  And 分页信息正确显示

Scenario: 按授权对象层级多选查询
  Given 用户在授权书列表页
  When 用户在"授权对象层级"下拉框选择"层级1"和"层级2"
  And 用户点击"查询"按钮
  Then 系统返回包含所选层级的授权书列表

Scenario: 按适用区域树形多选查询
  Given 用户在授权书列表页
  When 用户在"适用区域"下拉框选择"机关"下的"地区部1"的"代表处1"的"办事处1"
  And 用户勾选该节点
  And 用户点击"查询"按钮
  Then 系统返回适用区域包含所选节点的授权书列表

Scenario: 按状态查询
  Given 用户在授权书列表页
  When 用户在"状态"下拉框选择"草稿"
  And 用户点击"查询"按钮
  Then 系统返回状态为"草稿"的授权书列表

Scenario: 重置查询条件
  Given 用户在授权书列表页
  And 已输入多个查询条件
  When 用户点击"重置"按钮
  Then 所有查询条件清空
  And 表格显示全部数据（第一页）

Scenario: 分页查询
  Given 用户在授权书列表页
  And 列表数据超过10条
  When 用户切换页码或修改每页条数（10/30/50）
  Then 表格显示对应页的数据
  And 序号字段根据分页自动计算
```

##### 查询条件字段
| 字段名 | 类型 | 查询方式 | 数据来源 |
|--------|------|----------|----------|
| 授权书名称 | 单行文本输入框 | 模糊查询 | 用户输入 |
| 授权对象层级 | 下拉列表（平铺） | 多选精确匹配 | 公司、BU、营销服、地区部、代表处 |
| 适用区域 | 下拉列表（四层树形） | 层级多选 | 机关-地区部-代表处-办事处 |
| 授权发布层级 | 下拉列表（平铺） | 多选精确匹配 | 公司、BU、营销服、地区部、代表处 |
| 授权发布组织 | 下拉列表（四层树形） | 层级多选 | 机关-地区部-代表处-办事处 |
| 授权书发布年份 | 日期组件（年份） | 精确匹配 | 用户选择 |
| 状态 | 下拉列表 | 精确匹配 | 固定值：草稿/已发布/已失效 |

#### US-009: 配置场景规则详情

**以便于** 实现精准的场景匹配

**优先级**: Must Have
**故事点**: 13

##### 验收标准
```gherkin
Scenario: 场景配置弹窗基本字段
  Given 用户点击"添加场景"或"编辑"按钮
  Then 弹窗显示以下必填字段：

  And 场景名称、产业、业务场景三个字段一行
  And 决策层级单独一行
  And 具体规则与产业+业务场景对齐

Scenario: 新增规则条件
  Given 用户在场景配置弹窗
  And 规则配置区域为空白
  When 用户点击"[+新增条件]"文本按钮
  Then 系统新增一行规则条件
  And 显示规则字段下拉框
  And 文本按钮自动下移

  Given 用户在场景配置弹窗

  Then 系统在上一条件下方新增一行
  And 两个条件默认用"[且]"连接
  And 点击"[且]"可实现与"[或]"的互换

Scenario: 新增子条件组
  Given 用户在场景配置弹窗
  And 已配置规则条件
  When 用户点击"[+新增子条件组]"文本按钮
  Then 系统在下方新增一个方框
  And 方框内默认有一个规则字段下拉框
  And 方框下方显示三个文本按钮：[+新增条件]、[+新增子条件组]、[移除条件组]
  And 整个方框与上一条规则字段左对齐
  And 默认用"[且]"连接

Scenario: 删除规则条件

  Given 用户在配置规则条件
  When 用户选择对比目标类型"对比字段"

```
┌─────────┐    发布    ┌───────────┐    失效    ┌─────────┐
│  草稿   │ ────────> │  已发布   │ ────────> │  已失效  │
│ DRAFT  │           │ PUBLISHED │           │ INVALID │
└─────────┘           └───────────┘           └─────────┘
     │                      │                      │
     │ 删除                 │ 删除                 │ 删除
     ▼                      ▼                      ▼
  (已删除)               (已删除)               (已删除)

| 状态 | 可编辑 | 可保存 | 可发布 | 可失效 | 可删除 |
|------|--------|--------|--------|--------|--------|
| 草稿 | ✓ | ✓ | ✓ | ✗ | ✓ |
| 已发布 | ✗ | ✗ | ✗ | ✓ | ✓ |
| 已失效 | ✗ | ✗ | ✗ | ✗ | ✓ |

### 4.3 规则参数状态
- 生效（ACTIVE）：可在规则配置中使用
- 失效（INACTIVE）：不可在规则配置中使用

| 授权发布组织 | AUTH_PUBLISH_ORG | 四层树形，多选 | 机关-地区部-代表处-办事处（上级维度），调用服务获取 |
| 产业 | INDUSTRY | 树形，多选 | LV1-LV2-LV3（暂用层级代码），调用服务获取 |
| 业务场景 | BUSINESS_SCENE | 平铺列表，单选 | 调用服务获取 |
| 决策层级 | DECISION_LEVEL | 平铺列表，单选 | 调用服务获取 |
| 文档类型 | DOC_TYPE | 平铺列表，单选 | Lookup配置 |
| 状态 | STATUS | 平铺列表，单选 | 草稿/已发布/已失效 |
| 运算符 | OPERATOR | 平铺列表，单选 | >, <, =, >=, <=, != |
| 计量单位 | COMPARE_UNIT | 平铺列表，单选 | 千万、兆、CNY、USD等 |
| 对比类型 | COMPARE_TYPE | 平铺列表，单选 | 对比字段/数值/文本/比例 |
| 语言 | LANGUAGE | 平铺列表，单选 | 中文/英文 |
| 规则参数字段 | - | 平铺列表，单选 | 调用规则参数接口获取 |
| 是否生效 | IS_ACTIVE | 平铺列表，单选 | 是/否 |
| 数据类型 | DATA_TYPE | 平铺列表，单选 | 文本/数值/比对字段/比率 |

### 6.1 层级字段说明

**授权对象层级 vs 授权发布层级**：
- 层级值相同：公司、BU、营销服、地区部、代表处
- 业务含义差异：授权发布层级=上级（授权方所属层级），授权对象层级=下级或更下级（被授权方所属层级）

**适用区域 vs 授权发布组织**：
- 维度相同：机关、地区部、代表处、办事处（四层树形）
- 业务含义差异：授权发布组织=上级（发布授权的组织机构），适用区域=下级（授权适用的地理范围）

## 7. 非功能性需求

### 7.1 性能要求
- 列表查询响应时间 < 2秒
- 弹窗打开时间 < 1秒
- 分页默认每页10条，支持10/30/50选项

### 7.2 安全要求
- 所有操作需记录操作日志（创建、编辑、发布、失效、删除）
- 删除操作需二次确认
- 机机认证暂不实现，当前只实现接口

### 7.3 兼容性要求
- 支持主流浏览器（Chrome、Firefox、Edge）
- 响应式设计，支持1920x1080及以上分辨率
