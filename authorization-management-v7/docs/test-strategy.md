## 3. 测试类型

### 3.1 测试类型矩阵

| 测试类型 | 覆盖范围 | 执行阶段 | 测试方法 |
|----------|----------|----------|----------|
| 功能测试 | 全部用户故事 | 系统测试 | 手工 + 自动化 |
| 接口测试 | 34个API接口 | 系统测试 | 自动化 |
| 状态流转测试 | 授权书状态流转 | 系统测试 | 手工 |
| 业务规则测试 | 17条业务规则 | 系统测试 | 手工 + 自动化 |
| UI测试 | 前端页面布局、交互 | 系统测试 | 手工 |
| 异常测试 | 异常场景、边界值 | 系统测试 | 手工 |
| 回归测试 | 全部功能 | 发布前 | 自动化 |

### 3.2 功能测试详细说明

**测试重点**：
- 用户故事验收标准验证
- 页面交互流程完整性
- 字段显示和编辑状态控制
- 搜索条件组合查询
- 批量操作功能

### 3.3 接口测试详细说明

**测试重点**：
- 请求参数校验（必填、类型、范围）
- 响应结构校验（字段、格式、状态码）
- 业务错误码校验（1001-1008）
- 分页参数和响应
- 批量操作响应结构

### 3.4 状态流转测试详细说明

**授权书状态流转图**：
```
┌─────────┐    发布    ┌───────────┐    失效    ┌─────────┐
│  草稿   │ ────────> │  已发布   │ ────────> │  已失效  │
│ DRAFT  │           │ PUBLISHED │           │ INVALID │
└─────────┘           └───────────┘           └─────────┘
     │                      │                      │
     │ 删除                 │ 删除                 │ 删除
     ▼                      ▼                      ▼
  (已删除)               (已删除)               (已删除)
```

**测试重点**：
- 状态转换路径正确性
- 各状态下操作权限控制
- 状态校验错误提示

## 5. 测试方法

### 5.1 手工测试方法

| 方法 | 适用场景 | 执行方式 |
|------|----------|----------|
| 探索性测试 | UI交互、用户体验 | 自由探索 |
| 场景测试 | 业务流程验证 | 按用户故事验收标准执行 |
| 状态流转测试 | 状态机验证 | 按状态图路径执行 |
| 边界值测试 | 字段长度、数值范围 | 边界值+边界值±1 |
| 错误猜测测试 | 异常场景 | 基于经验猜测异常情况 |

### 5.2 自动化测试方法

| 方法 | 适用场景 | 工具 |
|------|----------|------|
| API接口测试 | 34个接口验证 | JMeter / Postman / Python+Requests |
| 数据校验测试 | 响应数据结构验证 | JSON Schema校验 |
| 回归测试 | 发布前全量验证 | 自动化脚本 |

### 5.3 测试用例设计原则

1. **覆盖性**：每个用户故事至少设计正向和逆向测试用例
2. **独立性**：测试用例可独立执行，不依赖其他用例结果
3. **可重复性**：测试用例可重复执行，结果一致
4. **清晰性**：用例描述清晰，步骤明确，预期结果具体

## 8. 测试计划

### 8.1 测试阶段划分

| 阶段 | 内容 | 输出物 |
|------|------|--------|
| 第一阶段 | 测试策略制定 | 测试策略文档 |
| 第二阶段 | 测试用例编写 | 功能测试用例、接口测试用例 |
| 第三阶段 | 测试执行 | 测试报告、缺陷列表 |
| 第四阶段 | 回归测试 | 回归测试报告 |

### 8.2 测试执行顺序

按照测试优先级执行：

**P0级别测试顺序**：
1. 授权书列表查询功能
2. 授权书创建、编辑、删除
3. 授权书状态流转（发布、失效）
4. 场景配置功能
5. 场景匹配接口

**P1级别测试顺序**：
6. 附件管理
7. 规则参数配置
8. 问卷管理
9. 场景规则详情配置

**P2级别测试顺序**：
10. 下拉数据接口
11. 操作日志功能

## 9. 缺陷管理

### 9.1 缺陷等级定义

| 等级 | 定义 | 处理时限 |
|------|------|----------|
| 致命 | 系统崩溃、数据丢失、核心功能无法使用 | 立即修复 |
| 严重 | 主要功能异常、状态流转错误 | 24小时内修复 |
| 一般 | 次要功能异常、UI显示问题 | 72小时内修复 |
| 轻微 | 建议性问题、不影响功能使用 | 发布前修复 |

### 9.2 缺陷记录格式

```
缺陷编号：BUG-XXX
缺陷标题：[模块名] 缺陷简要描述
缺陷等级：致命/严重/一般/轻微
发现阶段：系统测试
发现时间：YYYY-MM-DD
发现人：测试工程师
缺陷描述：详细描述问题现象
重现步骤：
1. 步骤1
2. 步骤2
3. 步骤3
预期结果：正常情况下的预期结果
实际结果：实际发生的结果
附件：截图或日志文件
```

## 11. 风险与对策

### 11.1 测试风险识别

| 风险 | 影响 | 对策 |
|------|------|------|
| 测试环境不稳定 | 测试无法执行 | 准备备用环境，及时反馈 |
| 测试数据不完整 | 测试覆盖不全 | 自行准备测试数据 |
| 需求变更频繁 | 测试用例需更新 | 及时沟通，快速响应 |
| 时间紧迫 | 测试覆盖不足 | 优先执行P0测试 |

### 11.2 质量风险识别

| 风险 | 影响 | 对策 |
|------|------|------|
| 规则引擎复杂 | 嵌套条件组逻辑复杂 | 重点测试规则匹配场景 |
| 状态流转控制 | 权限控制易出错 | 状态流转专项测试 |
| 多语言支持 | 问卷中英文切换 | 多语言组合测试 |
